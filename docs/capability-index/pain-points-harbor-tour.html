The Problem & The Solution

Without ContextCore

  • Developers update Jira, GitHub, and Slack — same info 3 places
  • PMs spend 2 hours every Monday compiling status
  • Engineering leaders open 5 tools to see portfolio state
  • SREs get 2am alerts with no service context
  • Compliance officers spend weeks assembling audit evidence
  • AI agents forget everything between sessions

With ContextCore

  • Status derived automatically from Git activity
  • Real-time dashboards replace manual compilation
  • Single portfolio view in Grafana
  • Alerts enriched with business context and ownership
  • Complete audit trail queryable via Loki
  • Agent insights persist as OTel spans

Estimation Methodology

Three methods, explicit confidence levels. Order of magnitude, not forecasts.

Time-Based

High confidence (±30%)
headcount × frequency × hours × rate
Used for: Redundant effort, context fragmentation, trust deficit

Risk-Based

Medium confidence (±50%)
frequency × P(event) × impact
Used for: Delayed feedback, crisis friction, downtime

Opportunity-Based

Low confidence (±3-5x)
capability_delta × adoption_drag
Used for: Blind spots, interoperability tax, agent pains

How to Use This Manifest

1. Understand Persona Pain Burden

Look at persona_roi_summary for each persona. Sort by annual cost to prioritize which persona to serve first.

2. Prioritize Roadmap by ROI

Use roi_rollup.gap_roi_ranking to find the highest-value gaps. #1 gap: Agent decision visibility ($275K, small effort).

3. Communicate Value to Audiences

Each pain point is written in the persona's voice with emotional weight. Use their language for sales and investor conversations.

4. Assess Delivered Value

The by_delivery_status section shows 56% of pain already addressed. Track this as benefits move from gap to delivered.

5. Adjust for Your Org

Replace org_profiles values with your headcounts and rates. Recompute using the documented formulas per pain point.

6. Track Pain Reduction Over Time

As benefits ship, update by_delivery_status. Pain points stay (documenting what WAS painful); only the rollup changes.
