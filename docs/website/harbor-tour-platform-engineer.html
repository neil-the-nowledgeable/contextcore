Value-Based Observability

ContextCore introduces the ProjectContext CRD - a single source of truth that drives observability configuration from business metadata.

apiVersion: contextcore.io/v1 kind: ProjectContext metadata: name: checkout-service namespace: commerce spec: project: id: "commerce-platform" epic: "EPIC-42" business: criticality: critical # Drives alert severity value: revenue-primary # Drives sampling rate owner: commerce-team # Drives routing requirements: availability: "99.95" # Generates SLO alerts latencyP99: "200ms" # Generates latency alerts design: doc: "https://docs.internal/checkout" adr: "ADR-015" targets: - kind: Deployment name: checkout-service

What Gets Generated

Project Signal Generated Artifact
business.criticality: critical 100% trace sampling, 10s metric scrape, page severity
requirements.latencyP99: 200ms PrometheusRule with histogram_quantile(0.99, ...) > 0.2
requirements.availability: 99.95 SLO burn rate alerts with appropriate windows
design.doc Runbook link in alert annotations
business.owner Alert routing to team's Slack/PagerDuty

Quick Start: Kubernetes

1

Install the CRD and Controller

Deploy ContextCore to your cluster

# Install via Helm helm repo add contextcore https://contextcore.io/charts helm install contextcore contextcore/contextcore \ --namespace contextcore-system \ --create-namespace # Or via kustomize kubectl apply -k github.com/contextcore/contextcore/k8s/observability
2

Create a ProjectContext

Define business context for your service

cat <<EOF | kubectl apply -f - apiVersion: contextcore.io/v1 kind: ProjectContext metadata: name: my-service namespace: default spec: business: criticality: high owner: my-team requirements: latencyP99: "500ms" targets: - kind: Deployment name: my-service EOF
3

Verify Generated Resources

Check what the controller created

# View generated ServiceMonitor kubectl get servicemonitor -l contextcore.io/project=my-service # View generated PrometheusRule kubectl get prometheusrule -l contextcore.io/project=my-service # Check annotations on target kubectl get deployment my-service -o jsonpath='{.metadata.annotations}'

CLI for Operators

# Install the CLI pip3 install contextcore # Verify installation (self-monitoring) contextcore install verify # Provision dashboards to Grafana contextcore dashboards provision --grafana-url http://grafana:3000 # Check project health contextcore metrics summary --project checkout-service # Operational commands contextcore ops doctor # Preflight checks contextcore ops health # Component status contextcore ops smoke-test # Full stack validation contextcore ops backup # Export Grafana state
