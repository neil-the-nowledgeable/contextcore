Agents That Remember

Build AI agents with persistent memory, observable execution, and cross-session context using the Wayfinder suite. Your agents learn from yesterday's sessions.

The Amnesia Problem

Every agent session starts from zero. Decisions made, lessons learned, context gathered—all lost when the session ends.

# Session 1 (Monday)
Claude: "I'll use async/await for this - it's cleaner than callbacks"
       [writes implementation, explains rationale, session ends]

# Session 2 (Tuesday)
Claude: "Let me understand this codebase..."
       [re-discovers the same patterns]
       "Should we use async or callbacks here?"
User:   "We discussed this yesterday!"
Claude: "I apologize, I don't have memory of previous sessions."

# Session 3 (Wednesday)
[Same conversation repeats...]

The cost: Re-explaining context, repeated decisions, lost institutional knowledge. Every session is groundhog day.

Agent Insights: Persistent Memory

ContextCore stores agent insights as OpenTelemetry spans in Tempo. Query them across sessions, agents, and time.

from contextcore.agent import InsightEmitter, InsightQuerier # Emit a decision (persists to Tempo) emitter = InsightEmitter(project_id="checkout", agent_id="claude") emitter.emit_decision( summary="Using async/await for cleaner API handlers", confidence=0.92, context={"file": "src/api/handlers.py"} ) # Query prior decisions (in future sessions) querier = InsightQuerier() prior = querier.query( project_id="checkout", insight_type="decision", time_range="30d" ) for decision in prior: print(f"{decision.summary} (confidence: {decision.confidence})")

Insight Types

Type Use Case Example
decision Architectural choices, design rationale "Chose Redis over Memcached for TTL support"
lesson Patterns learned, gotchas discovered "Always mock OTLP exporter in unit tests"
question Open items needing human input "Should we support OAuth 2.1?"
handoff Context passed between agents "Investigation complete, ready for implementation"

Indexed Capabilities

ContextCore capabilities are indexed in machine-readable format for agent discovery. Each capability has maturity level, triggers, and API documentation.

Capability Maturity Trigger
insight.emit stable After making decisions, learning patterns
insight.query stable Start of session, before decisions
handoff.initiate beta Task exceeds scope, needs specialist
handoff.receive beta Accepting work from another agent
guidance.read beta Checking human constraints/preferences
code_generation.contract beta Before generating code, agree on constraints

Agent-to-Agent Handoffs

Structured handoffs with context preservation. The receiving agent knows what was tried, what worked, and what the user expects.

┌─────────────────────────────────────────────────────────────────────┐
│                    Multi-Agent Workflow                             │
└─────────────────────────────────────────────────────────────────────┘

Investigator Agent (Claude)
    │ Analyzes error logs, identifies root cause
    │ Emits: insight.decision ("Found race condition in cart.py:142")
    │
    ▼
Handoff Span ──────────────────────────────────────────────────────
    │ handoff.from_agent: "investigator"
    │ handoff.to_agent: "designer"
    │ handoff.context: { root_cause, affected_files, severity }
    │ handoff.expected_output: "Fix approach with test strategy"
    │
    ▼
Designer Agent (GPT-4)
    │ Proposes fix approach, identifies test cases
    │ Emits: insight.decision ("Use optimistic locking pattern")
    │
    ▼
Handoff Span ──────────────────────────────────────────────────────
    │
    ▼
Implementer Agent (Gemini)
    │ Writes code, runs tests
    │ Emits: insight.lesson ("Optimistic locking needs retry logic")
    │
    ▼
┌─────────────────────────────────────────────────────────────────────┐
│  All spans visible in Tempo │ Queryable via TraceQL                 │
└─────────────────────────────────────────────────────────────────────┘
from contextcore.agent import HandoffManager # Initiate a handoff handoff = HandoffManager(project_id="checkout", agent_id="investigator") handoff.initiate( to_agent="designer", capability_id="fix_design", context={ "root_cause": "Race condition in cart update", "affected_files": ["src/cart/service.py"], "severity": "high" }, expected_output="Fix approach with test strategy" ) # Receive and process handoff pending = handoff.receive_pending(agent_id="designer") for task in pending: print(f"From: {task.from_agent}, Context: {task.context}")

Code Generation Contracts

Prevent truncation before it happens. Agree on output constraints before generation.

from contextcore.agent.code_generation import ( CodeGenerationHandoff, CodeGenerationSpec, ) # Request code with size constraints handoff = CodeGenerationHandoff(project_id="myproject", agent_id="orchestrator") result = handoff.request_code( to_agent="code-generator", spec=CodeGenerationSpec( target_file="src/mymodule.py", description="Implement CartService class", max_lines=150, # Size constraint required_exports=["CartService"], # Completeness markers ) ) # If size exceeds limit, decomposition is negotiated via span events # Proactive prevention, not reactive validation

OTel GenAI Semantic Conventions

The ContextCore standard aligns with OTel GenAI semantic conventions. Wayfinder's dual-emit mode sends both legacy (agent.*) and standard (gen_ai.*) attributes during migration.

Quick Start

# Install ContextCore pip3 install contextcore # Install with LLM provider support pip3 install contextcore-beaver # Formerly startd8 # Set up local storage for development export CONTEXTCORE_LOCAL_STORAGE=~/.contextcore/insights # Or point to Tempo for production export OTEL_EXPORTER_OTLP_ENDPOINT=localhost:4317
# Basic usage in your agent code from contextcore.agent import InsightEmitter, InsightQuerier # At session start: load prior context querier = InsightQuerier() prior_decisions = querier.query( project_id="my-project", insight_type="decision", time_range="30d" ) # During work: emit insights emitter = InsightEmitter(project_id="my-project", agent_id="claude") emitter.emit_decision( summary="Chose approach X because Y", confidence=0.9 ) # At session end: persist lessons emitter.emit_lesson( summary="Always validate inputs before processing", category="best-practice", applies_to=["src/api/"] )
